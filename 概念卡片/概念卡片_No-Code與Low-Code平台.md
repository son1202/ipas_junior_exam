# 🚀 概念卡片_No-Code與Low-Code平台

## 📝 介紹與詳細說明

### 🟢 初學者/非技術視角 (跨部門溝通)
為了讓「不懂寫程式的人也能做軟體」，市場上出現了兩種神兵利器，讓開發 APP 或網頁變得像「玩樂高積木」一樣簡單：
1. **No-Code (完全無程式碼)**：
   - **概念**：真的是純拖拉點拽！你只需要把需要的按鈕、表格、圖片從旁邊拉進畫面裡，設定好規則就能用。
   - **適合對象**：完全不懂寫程式的行銷、人資或業務人員。
   - **優缺點**：超快、超簡單，但「樂高積木長什麼形狀，你就只能拼什麼」，變化比較受限。例如：行銷部用 Wix 兩小時就架好美美的活動網頁。
2. **Low-Code (低程式碼)**：
   - **概念**：大部分介面也是用拖拉的，但遇到比較特殊的需求（例如要連上公司自己古老的資料庫時），允許工程師在裡面寫個幾行簡短的程式碼來打通關節。
   - **適合對象**：IT 部門或是懂一點程式邏輯的進階使用者。
   - **優缺點**：兼備了「快速」與「彈性客製化」。工程師不用再從零開始造輪子，開發內部報帳系統的時間可以縮短一半以上。

### 🔵 深度專業視角 (核心技術與底層邏輯)
No-Code / Low-Code (NCLC) 平台本質上是高度抽象化的模型驅動架構 (Model-Driven Architecture) 與視覺化編程 (Visual Programming) 的具現：
1. **技術實現**：
   - 將常見的 UI 元件、CRUD 資料庫操作、商業邏輯 (Business Logic) 封裝為可重用的模塊或 JSON 設定檔。
   - 透過平台的運行環境 (Runtime Engine)，將視覺化的配置即時直譯 (Interpret) 或轉譯成目標平台的底層代碼 (如 React, Node.js 等)。
2. **與 AI 的結合 (Generative UI)**：
   - 現代平臺深度整合了 LLM。使用者只需用文字給出 Prompt (例如：「幫我生成一個客戶投訴表單，包含三個文字框跟一個圖表」)，AI 即可直接寫出 JSON 結構並渲染成 NCLC 平台的 UI 框架。
3. **企業級考量與挑戰**：
   - **Vendor Lock-in (平台綁定)**：一旦依賴特定平台，轉移成本極高，因專有組件代碼並不互通。
   - **Shadow IT (影子 IT)**：非 IT 單位若私自開發大量微型應用，容易發生權限控管疏漏與資料孤島，必須建立完善的平台治理 (Governance) 守則。

## 🎯 常見測驗題型

## 🎯 常見測驗題型

> [!question] 選擇題 1：
> 某家中小企業的行銷主管，完全沒有學過寫程式，但他想在下週前快速建立一個收集客戶滿意度的網頁表單與抽獎小遊戲。下列哪一種工具最適合他的需求？
> (A) 使用 Python 進行全端開發
> (B) Low-Code 低程式碼平台
> (C) No-Code 無程式碼平台
> (D) Linux 終端機指令
> 
> **解答**：(C)。No-Code 完全針對零程式基礎的人員設計，用拖拉方式即可快速建立標準化應用。

> [!question] 選擇題 2：
> 相較於傳統由工程師一行一行撰寫程式碼的開發方式，IT 部門導入 Low-Code 平台的主要優勢為何？
> (A) 系統執行速度保證會比傳統程式碼快十倍
> (B) 平台具備強大的圖形化基礎，大幅縮短常規開發時間，並容許工程師寫一點 code 進行複雜系統的連接客製化
> (C) 能夠完全解雇並取代所有專業軟體工程師
> (D) 完全無需考慮任何資訊安全風險
> 
> **解答**：(B)。Low-Code 兼具「減少重複造輪子的速度」與「保留少許寫 code 彈指擴展的能力」的雙重優勢。

> [!question] 選擇題 3：
> 公司的系統非常老舊，是在 20 年前開發的 DOS 畫面，完全沒有提供現代化的 API 給其他系統自動撈取資料。此時若想讓 No-Code 平台能夠打通這個舊系統，最常搭配哪一種技術，讓電腦去「模擬人手敲鍵盤」自動抓取畫面上的數字？
> (A) API 應用程式介面
> (B) Webhook 網頁回呼
> (C) 迴積神經網路 (CNN)
> (D) RPA (機器人流程自動化)
> 
> **解答**：(D)。RPA 專門用來針對無 API 可用的遺留系統 (Legacy Systems) 進行螢幕抓取與自動化操控。

> [!question] 選擇題 4：
> 在系統串接時，我們常說「主動去向別人要資料」和「發生事件時，別人主動推播過來」是兩種截然不同的設計。請問「外部系統刷卡成功時，主動發送通知推播到我們平台」的機制稱為？
> (A) Webhook (網頁回呼)
> (B) RESTful API 發起請求
> (C) SQL Select 查詢
> (D) K-Means 演算法
> 
> **解答**：(A)。Webhook 本質上就是反向的 API，是由事件發生端主動向接收端推播狀態，即時性非常高。

> [!question] 選擇題 5：
> 企業內部各業務單位若隨意使用市面上的各種免費 No-Code 平台自行開發小系統，雖然提升了業務效率，但也可能讓 IT 部門失去掌控，造成資料散落且有嚴重資安隱患。這個現象在企業管理中通常被稱為什麼？
> (A) 神經網路過擬合 (Overfitting)
> (B) 影子 IT (Shadow IT)
> (C) 區塊鏈算力浪費
> (D) A/B 測試差異
> 
> **解答**：(B)。影子 IT 是指在 IT 部門的視野與管控範圍之外，員工私自採用的軟硬體解決方案。

> [!question] 選擇題 6：
> 如果一家企業極度重視系統底層的所有權 (Ownership)，且未來隨時需要把整個系統的程式碼搬移更換到完全不同架構的雲端伺服器上。此時若他們全面採用市面綁定的 No-Code 平台作為核心服務，可能會面臨最大的風險是？
> (A) 網路線路會加速老化
> (B) 廠商鎖定 (Vendor Lock-in)，因為自己無法取出具體的原始碼來轉移
> (C) AI 引擎會產生幻覺
> (D) 必須額外花費大量算力去優化神經網路
> 
> **解答**：(B)。依賴封閉式的 No-Code 平台，一旦想轉換跑道或平台倒閉，往往只能重頭開發。
