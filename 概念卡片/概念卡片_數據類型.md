# 📊 概念卡片_數據類型

## 📝 介紹與詳細說明

### 🟢 初學者/非技術視角 (跨部門溝通)
在讓 AI 變聰明之前，我們得先餵給它各種「資料」當作教材。而這些資料依照整齊程度，可以分成三種：
1. **結構化數據 (就像 Excel 報表)**：最乖、最守規矩的資料。它們排成整齊的表格，有明確的行與列，像是銀行的客戶交易紀錄、公司的產品庫存表。傳統電腦程式最擅長處理這種資料。
2. **半結構化數據 (就像夾帶標籤的筆記)**：沒有排成死板的表格，但還是有規律可循。例如我們常常在網頁上看到的 HTML 代碼，或是工程師最愛用的 JSON 檔案。它用各種「標記符號」把不同的內容分隔開來。
3. **非結構化數據 (就像一個大雜燴)**：完全沒有固定格式，也是目前世界上最多（佔 80% 以上）的資料類型。例如你的自拍照、YouTube 影片、通話錄音或是 PDF 說明書。這種資料過去很難被利用，但現在剛好是 AI (特別是生成式 AI) 最拿手的領域。

### 🔵 深度專業視角 (核心技術與底層邏輯)
資料工程是 AI 訓練 Pipeline 的第一關。從 Schema (資料結構定義) 嚴謹度的維度，數據可分為：
1. **結構化數據 (Structured Data)**：
   - **特徵**：具備嚴格定義的 Schema-on-write，遵循關聯式與集合代數。
   - **儲存與處理**：儲存於 RDBMS 或資料倉儲中。通常支援嚴密的 SQL 語法進行複雜的關聯 (JOIN) 運算，確保交易的 ACID (原子性、一致性、隔離性、持久性) 特性。
2. **半結構化數據 (Semi-structured Data)**：
   - **特徵**：自我描述 (Self-describing) 的資料結構，通常是 Schema-less 或是 Schema-on-read。透過階層式的標籤或鍵值對 (Key-Value) 定義語義。
   - **儲存與處理**：常駐於 NoSQL 資料庫 (如 MongoDB, Redis) 中，典型序列化傳輸格式為 JSON, XML，或欄位式儲存如 Parquet。極適合微服務架構間的 RESTful API 輕量化傳輸。
3. **非結構化數據 (Unstructured Data)**：
   - **特徵**：缺乏預先定義的資料模型，本質上是連續的二進位位元流 (Binary Streams) 或高頻率序列。
   - **儲存與處理**：依賴物件儲存體 (Object Storage) 或 Hadoop HDFS 進行海量、非結構化的歸檔（形成 Data Lake）。後續需要透過機器學習框架，利用特徵提取器 (如 CNN 或 Transformer )，將其降維並轉換為可被計算神經網路吸收的連續高維向量 (Embeddings)。

## 🎯 常見測驗題型

> [!question] 選擇題 1：
> 電子商務網站透過 API 將每天的訂單資訊以 JSON 格式匯出，以供其他系統使用。請問 JSON 格式主要屬於哪一種數據類型？
> (A) 結構化數據
> (B) 半結構化數據
> (C) 非結構化數據
> (D) 複合數據
> 
> **解答**：(B)。JSON 具有標籤與層級結構（鍵值對），但不如關聯式資料庫嚴格，故列為半結構化。

> [!question] 選擇題 2：
> 醫院為了訓練醫療 AI，收集了十萬張胸部 X 光照片與醫師口述的錄音檔。這批資料屬於下列何種數據類型？
> (A) 結構化數據
> (B) 半結構化數據
> (C) 非結構化數據
> (D) 時間序列數據
> 
> **解答**：(C)。影像、音訊與自由文本皆無法輕易寫入傳統表格欄位，屬於非結構化數據。

> [!question] 選擇題 3：
> 關於「資料湖 (Data Lake)」與「資料倉儲 (Data Warehouse)」的差異，下列敘述何者錯誤？
> (A) 資料倉儲主要存放已經清洗、具備明確 Schema 的結構化資料
> (B) 資料湖可以任意堆放結構化、半結構化與非結構化原始資料
> (C) 資料倉儲適合提供給 BI (商業智慧) 工具產出高層月報表
> (D) 資料湖具備強大的 ACID 交易特性，是現代銀行做金流結算的核心系統
> 
> **解答**：(D)。資料湖主要擅長海量資料儲存與大數據探勘，而不會拿來取代需要強烈交易一致性 (ACID) 的關聯式資料庫。

> [!question] 選擇題 4：
> 企業將龐大的伺服器 Log 檔 (紀錄了使用者 IP、點擊時間與瀏覽頁面標籤) 一筆筆寫入至 NoSQL 分散式資料庫中。這類 Log 檔案最適合被歸類為哪一種資料型態？
> (A) 非結構化數據
> (B) 半結構化數據
> (C) 強度關聯結構化數據
> (D) 靜態固定據點數據
> 
> **解答**：(B)。Log 檔通常帶有特定規則的標記符號與屬性，但不像 RDBMS 有死板的表格限制，屬於半結構化應用。

> [!question] 選擇題 5：
> 在現代化的大數據處理管線中，「ELT」架構逐漸興起。請問「ELT」架構的主要特點為何？
> (A) 在匯出 (Extract) 後立刻在本機端運算轉換 (Transform)，減輕目的地資料庫負擔
> (B) 先執行繁重的人工標記，再去撈取資料
> (C) 先將原始資料匯入 (Load) 資料湖，等後續需要時再透過強大的雲端算力進行轉換 (Transform)
> (D) 完全不需要進行資料清理，由 AI 模型自行除錯
> 
> **解答**：(C)。現代 ELT 架構仰賴資料湖的包容性與強大雲端叢集算力，因此選擇先 Load 後 Transform。

> [!question] 選擇題 6：
> 一份傳統企業級的 ERP (企業資源規劃) 系統，裡面儲存了數十萬筆包含「訂單編號、單字總額、出貨日期」的資料列。這屬於哪一種資料類型？
> (A) 結構化數據
> (B) 半結構化數據
> (C) 非結構化數據
> (D) 單一化數據
> 
> **解答**：(A)。具有高度組織化且可輕易以 Schema 定義的表格，屬於結構化數據。
