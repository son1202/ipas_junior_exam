# 生成考題操作指南 (Generate Exam Skill)

當使用者要求「生成考題」、「製作選擇題考卷」或是「依照概念卡片和專題討論產出模擬考卷」時，請嚴格遵守以下格式與設計規範，產出適合列印與填答的試卷。

## 1. 考卷排版與標籤
- 考卷頂端必須包含 YAML frontmatter 標籤，例如：
  ```yaml
  ---
  tags:
    - 模擬考題
  ---
  ```
- 使用一級標題 (H1) 標示考卷名稱，加上適當的 Emoji，例如：`# 📝 模擬考卷 第一回`。

## 2. 題目本體格式 (重點！)
- 必須使用 Obsidian 的區塊引用機制 (Callout) 來包裹每一題。
- 題目的標題列必須嚴格標示為：`> [!question] [  ] 第 X 題`。
- **絕對要包含 `[  ]`**：這是提供給使用者作答填寫 (A/B/C/D) 的空白括號。
- 題目的情境敘述以及選項 (A/B/C/D) 都必須全部包覆在 `>` 區塊內。

**正確的產出範例：**
```markdown
> [!question] [  ] 第 1 題
> 這裡填寫題目敘述，例如某某演算法的機制為何？
> (A) 選項一
> (B) 選項二
> (C) 選項三
> (D) 選項四
```

## 3. 解答與解析隔離機制
- 試卷的靈魂在於測驗，**嚴禁**將解答直接暴露在題目的下方。
- 所有的解答與詳細原理說明，必須一次性地集中搬移到整份文件的最下方。
- 在所有題目出完之後，使用分隔線 `---`，並加上 `# 📝 考題解答` 作為解答區塊的大標題。
- 下方採用條列式或小標題來列出每一題的最佳解答與理由。

**正確的解答區範例：**
```markdown
---

# 📝 考題解答

### 第 1 題
**解答**：(B)。因為某某技術具備這些底層特性，所以答案為 B。

### 第 2 題
**解答**：(C)。這是因為在極端資料下這套演算法具備更高適應性...
```

## 4. 產出內容要求
- 若無法從既有文件的標籤中抽取考題，請自行從參考文本萃取高含金量知識點設計全新的選擇題，務必確保考題具有邏輯推理性與誘答力。
- 題目總數基本為50題，若使用者指定其他數量，則依使用者指定而定。
- 最後產出的檔名如果無特殊要求，請放置於該專案下的目錄 (如 `模擬考題\` 資料夾) 並妥善命名。
